## 1. 컬러 팔레트 스위처 — 시도 및 철회

### 시도한 방식
헤더 우측 토글 버튼 옆에 작은 팔레트 선택 버튼을 두고, 클릭 시 5가지 테마 컬러를 드롭다운으로 선택할 수 있도록 구현했다.

- CSS 커스텀 프로퍼티(`--color-primary` 등)를 루트에 동적으로 설정
- `ThemeContext`에 `palette` 상태 추가
- `PaletteButton` + `PaletteDropdown` 컴포넌트 신규 작성
- `tailwind.config.js`에서 `colors` 값을 CSS 변수 참조로 변경

### 문제
브라우저에서 팔레트 전환이 전혀 동작하지 않음.  
Tailwind의 JIT 컴파일러는 빌드 시점에 클래스를 정적으로 생성하므로, 런타임 CSS 변수 변경이 Tailwind 유틸리티 클래스에 반영되지 않는 구조적 한계가 있었다.

### 결정
팔레트 스위처 전면 제거. ThemeContext를 다크모드 전용으로 단순화.  
`tailwind.config.js`를 하드코딩 컬러 방식으로 원복.

## 3. Open Graph 메타 태그 + OG 이미지

### 메타 태그 (`index.html`)
URL 공유 시 SNS 미리보기가 올바르게 표시되도록 OG 태그 추가:

```html
<meta property="og:url"          content="https://eclipticwin.github.io/rest04/" />
<meta property="og:type"         content="website" />
<meta property="og:site_name"    content="AILearn" />
<meta property="og:title"        content="AILearn · AI 교육 플랫폼" />
<meta property="og:description"  content="AI 기초부터 실무 활용까지..." />
<meta property="og:image"        content="https://eclipticwin.github.io/rest04/og-image.png" />
<meta property="og:image:width"  content="1200" />
<meta property="og:image:height" content="630" />
<meta name="twitter:card"        content="summary_large_image" />
```

### OG 이미지 생성 (`scripts/generate-og.mjs`)
Node.js `sharp` 패키지(devDependency)로 1200×630 PNG 이미지 자동 생성.

- 배경: 다크 네이비 그라디언트 (`#0F1B2D` → `#1E3A8A`)
- 브랜드 컬러 5색 스와치 배치
- 한국어 헤드라인 + AILearn 로고 텍스트
- SVG 벡터를 sharp로 래스터화 → `public/og-image.png` 저장

```bash
node scripts/generate-og.mjs   # 재생성 시 실행
```

생성된 `public/og-image.png` 은 `git add`하여 GitHub Pages에 함께 배포.

## 4. 스크롤 Reveal 애니메이션

### 배경
chinhung.co.kr(rest03 원본)처럼 스크롤 시 요소가 자연스럽게 등장하는 효과를 요청받았으나 초기 구현에 누락되었다.  
rest03은 jQuery + `.on('scroll')` 방식으로 구현되어 있어 React에 직접 이식 불가능. 동일한 UX를 React 방식으로 재구현했다.

### 구조

#### `src/components/Reveal.jsx` — 신규 컴포넌트

```jsx
export default function Reveal({ children, className, delay, direction, threshold }) {
  // IntersectionObserver: 요소가 뷰포트에 12% 진입 시 'revealed' 클래스 추가
  // observer.unobserve() — 한 번 실행 후 해제 (반복 없음)
}
```

Props:
| Prop | 기본값 | 설명 |
|------|--------|------|
| `direction` | `'up'` | `up / left / right / fade / scale` |
| `delay` | `0` | CSS transitionDelay (ms), 카드 stagger 용도 |
| `threshold` | `0.12` | 뷰포트 진입 비율 |

#### `src/index.css` — 애니메이션 클래스

**히어로 입장 애니메이션** (페이지 로드 즉시):
```css
@keyframes heroFadeUp { from { opacity:0; transform:translateY(36px) } ... }
.hero-anim    { animation: heroFadeUp 0.85s cubic-bezier(0.22,1,0.36,1) forwards; }
.hero-d1~d5   { animation-delay: 0.05s ~ 0.72s; }  /* stagger */
```

**스크롤 Reveal 클래스**:
```css
.reveal-up    { opacity:0; transform: translateY(48px); }
.reveal-left  { opacity:0; transform: translateX(-52px); }
.reveal-right { opacity:0; transform: translateX(52px); }
.reveal-scale { opacity:0; transform: scale(0.90); }
.reveal-fade  { opacity:0; }
.revealed     { opacity:1 !important; transform:none !important; }
/* transition: 0.75s cubic-bezier(0.22,1,0.36,1) — easeOutQuint 유사 */
```

접근성: `prefers-reduced-motion: reduce` 설정 시 모든 애니메이션 즉시 비활성화.

#### 히어로 vs 스크롤 구분 이유
히어로 섹션은 페이지 최상단에 위치 → `IntersectionObserver` threshold 도달 불가.  
별도 `@keyframes + animation-delay` stagger로 처리.

### 페이지별 적용 전략

| 페이지 | 히어로 (`hero-anim`) | 섹션 타이틀 | 카드/아이템 |
|--------|---------------------|-------------|-------------|
| **Home** | `d1`배지 → `d2`제목 → `d3`설명 → `d4`버튼 → `d5`통계 | `<Reveal>` | `<Reveal delay={i*80~100} direction="up">` |
| **Videos** | `d1`카테고리 타이틀 → `d2`탭바 | `<Reveal>` | `<Reveal delay={i*80}>` |
| **About** | `d1`제목 → `d2`탭 | 탭별 `<Reveal>` | 방향별 stagger (left/right/up) |
| **Curriculum** | `d1`제목 → `d2`설명 | level 가이드 `<Reveal>` | 카테고리별 `delay={i*60}` |
| **Contact** | `d1`제목 → `d2`설명 | - | 연락처 `direction="left"`, 폼 `direction="right"` |

### 기술 선택 근거

**IntersectionObserver vs scroll 이벤트**
- `scroll` 이벤트: 매 프레임 실행 → 성능 저하, `throttle` 필요
- `IntersectionObserver`: 브라우저 네이티브 최적화, 메인 스레드 부담 없음, 모든 모던 브라우저 지원

**순수 CSS vs GSAP/Framer Motion**
- 외부 라이브러리 추가 없이 구현 → 번들 크기 불변
- `cubic-bezier(0.22, 1, 0.36, 1)` 베지어 커브로 자연스러운 감속 표현
- `will-change: opacity, transform` — GPU 가속 힌트
